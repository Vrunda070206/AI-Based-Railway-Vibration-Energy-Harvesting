Design a **modern, futuristic full-stack web dashboard** for a project titled:

**“AI-Based Railway Vibration Energy Harvesting Monitoring System”**

The system monitors multiple IoT nodes installed on railway tracks that generate electricity using piezoelectric discs and send real-time data.

---

## 🎯 Overall Design Style

* Dark theme UI (smart infrastructure style)
* Minimal, clean, professional layout
* Use glowing accents (electric blue, neon green, orange, red)
* Smooth animations and hover effects
* Responsive design (desktop + tablet)

---

## 🏠 1. Main Dashboard (Landing Page)

Create a dashboard with summary cards:

* Total Nodes Installed
* Active Nodes
* Nodes with Alerts
* Total Energy Generated Today
* System Health Status

Include:

* Small trend indicators (up/down arrows)
* Clean card-based layout

---

## 🔋 2. Energy Monitoring Section

Add visual graphs:

* Line chart: Energy generated over time
* Bar chart: Energy per node
* Battery-style indicators: Remaining energy per node
* Circular progress indicators for energy levels

---

## 📍 3. Railway Node Map

Design a visual railway track map showing nodes placed along it.

Each node should have color-coded status:

* Green → Healthy
* Yellow → Low Energy
* Red → Fault / Offline

On clicking a node, show a popup with:

* Node ID
* Energy Stored (%)
* Vibration Events Count
* Last Updated Time
* Status

---

## 🚨 4. Alerts & Notifications Panel

Create a panel showing alerts such as:

* Low Energy Warning
* Node Offline
* Abnormal Energy Generation
* Communication Failure

Each alert must show:

* Timestamp
* Severity (Info / Warning / Critical)
* Icon-based indicators

---

## 🤖 5. AI Insights Section

Add an AI analytics panel displaying:

* Predicted energy trends
* Estimated time before energy depletion
* Node reliability score
* Future failure prediction

Include small graphs and percentage indicators.

---

## 📊 6. Analytics Page

Advanced visualization:

* Energy generation trends over days/weeks
* Node comparison charts
* Peak vibration activity times
* Efficiency metrics

---

## 🧾 7. Activity Logs

Create a scrolling log panel showing:

* Train vibration detected
* Energy generated
* Alerts triggered
* Node communication updates

---

## ⚙️ 8. Node Management Panel

Include controls to:

* Add / remove nodes
* Configure alert thresholds
* View node configurations

---

## 👤 9. User Authentication

Include:

* Login page
* Role-based access (Admin / Viewer)
* Profile settings

---

## 🔁 10. Real-Time Data Behavior

Simulate real-time updates:

* Live energy values changing
* Alerts appearing dynamically
* Node status updating

---

## 🎨 UI Components to Include

* Cards
* Graphs (line, bar)
* Battery indicators
* Toggle switches
* Tables
* Notifications panel
* Sidebar navigation

---

## 📱 Navigation Layout

Left sidebar menu with sections:

* Dashboard
* Energy

* Alerts

* Analytics

* Settings

---

## 💡 Goal of the Design

The website should look like a **real-world smart infrastructure monitoring system used in railways or smart cities**, clearly showing:

* Energy harvesting performance
* System health
* AI-based predictions
* Real-time monitoring

## 🔥 Final Output Expectation

A **complete professional dashboard UI** that visually represents:

✔ IoT + AI system
✔ Energy monitoring
✔ Smart alerts
✔ Predictive analytics

The design should be impressive enough for a **final year engineering project or research presentation**.
